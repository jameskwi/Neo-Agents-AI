## What's New

### M3: Solution Architect Agent is live (`/neo:sa`)

The SA agent is now fully implemented. Run `/neo:sa` after completing a BRD with `/neo:ba` to get a full Solution Design Document (SSD) covering:

- API endpoints (method, path, auth, request/response)
- Data model changes (new tables, modified models, indices)
- New dependencies with version constraints and purpose
- Technical risks with likelihood, impact, and concrete mitigations
- An explicit `design_required: YES / NO` decision that routes you to `/neo:ds` or `/neo:dev`

The agent conducts a 3–5 question technical interview — one question at a time — before writing anything. It scans your codebase for routes, models, middleware, and dependencies so questions are specific to your actual stack.

Outputs saved to `.ai-agents/docs/SA/` and appended to the SPEC file. `tasks.json` updated automatically.

## Breaking Fixes (were silently broken)

**Plugin manifest was incomplete.**
The root `.claude-plugin/plugin.json` was missing the entire commands array, agents list, runtime block, and dashboard block. The plugin could not register any commands. Now complete with all 7 commands and full metadata.

**Inner plugin paths were wrong.**
`neo-agents-ai-plugin/.claude-plugin/plugin.json` had paths like `"skills/setup.md"` that resolved to the wrong directory. All paths corrected with `../` prefix.

**Directory naming was inconsistent.**
`SE` and `PL` were used in some files while `DEV` and `PM` were used in others — across `skills/setup.md`, `agents/engineer.md`, `CLAUDE.md`, `dashboard/server.js`, and `dashboard/index.html`. All now uniformly `DEV` / `PM`.

---

## Security

**Dashboard API is now token-protected.**
All `/api/*` routes require `?token=` matching the `access_token` generated by `/neo:setup`. The token is passed in the browser URL when the dashboard opens, persisted to `sessionStorage` so refreshes stay authenticated.

**CORS restricted to same-origin.**
Changed from `Access-Control-Allow-Origin: *` to `http://localhost:{port}` — dashboard API is no longer callable from other browser origins.

---

## Bug Fixes

**Python 3.8 compatibility restored.**
`datetime.UTC` (Python 3.11+ only) replaced with `datetime.timezone.utc` in the BA agent and SA agent. The declared minimum is Python 3.8.

**Dashboard timeline is now data-driven.**
Milestone statuses and phase percentages are derived from actual doc presence (`docs.BA`, `docs.SA`, etc.) and task completion counts — no more hardcoded dates or static percentages.

**Dashboard CDN disclaimer corrected.**
README previously claimed the dashboard required no internet. CDN loads (React, Babel, fonts) require internet on first use; browser-cached after that.
